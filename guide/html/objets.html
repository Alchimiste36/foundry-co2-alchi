Introduction

Il existe 6 sortes d'Objets dans le système Chroniques Oubliées Fantasy et chacun a son utilité spécifique.

Tous les Objets ont un onglet Description identique qui sera expliqué juste après.

Les types d'Objet sont :

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuille-profil]{Les PROFILS}

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuille-trait]{Les TRAITS}

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuile-voie]{Les VOIES}

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuille-capacite]{Les CAPACITÉS}

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuille-equipement]{Les ÉQUIPEMENTS}

  • @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.e46m0KDmGFAP5USg#feuille-attaque-(de-rencontre)]{Les ATTAQUES (DE RENCONTRE)}

Dans le Compendium COF2 Objets, vous pourrez trouver tous les objets présents dans le Livre de Règles de Chroniques Oubliées Fantasy v2.

Entête

Pour chaque type d'Objet, l'entête comporte les mêmes informations. La couleur de fond est différentes entre les types pour facilement repérer le type d'Objet que vous avez ouvert.

Image et Nom

Tous les Objets, quelque soit leur type, possèdent une image et un nom. Il est possible de les modifier en cliquant dessus.

Type

Le type de l'Objet est rappelé dans un rectangle en haut à droite de la fenêtre de l'Objet.

Onglet Description

Origine

Dans l'onglet Description, vous pouvez renseigner l'origine de l'Objet. S'il vient d'un livre de Chroniques Oubliées Fantasy, nous vous conseillions de renseigner également le numéro de la page pour pouvoir remonter à la source si vous avez un doute. Si l'Objet est inventé pour votre campagne, trouvez une dénomintaion personnelle pour savoir rapidement que cet Objet n'existe que dans votre univers. 

Description

Cet éditeur de texte permet d'écrire la description, les pouvoirs, les informations de l'Objet. Comme pour les journaux, par exemple, il est possible d'agrémenter votre texte avec des images, des tableaux, des listes… 

Si l'Objet possède une ou plusieurs Actions ou Effets configurés, renseignez-le dans la descrption pour ne pas perdre vos joueurs et leur permettre de comprendre comment utiliser la Capacité ou l'Équipement. Vous remarquerez que pour les Objets du Compendium du Livre de Règles, tous les Objets possédant une Action ou un Effet, un paragraphe "Usage en jeu" vous explique comment fonctionne l'Objet.

Verrouillage des feuilles

Les feuilles Objet possèdent un Slider en haut à gauche de la fenêtre. Ce Slider permet de gérer le Verrouillage de la feuille. Si vous ne voyez pas le Slider c'est que vous n'avez pas les droits assez hauts pour déverrouiller la feuille ou alors vous êtes en train de consulter un Objet présent dans un Compendium. 

Si l'Objet est dans un Compendium, il faudra d'abord l'importer pour pouvoir l'ouvrir et le déverrouiller pour modifier des valeurs de la feuille.

Lorsque la feuille est Déverrouillée, il est possible de modifier toutes les valeurs de l'Objet, de sélectionner des propriétés, supprimer des éléments... Il est fortement recommandé d'utiliser le plus souvent possible les Feuilles verrouillées afin de ne pas modifier ou supprimer des informations importantes de l'Objet par mégarde.

Mode Debug

Si vous avez activé dans les options du système le @UUID[JournalEntry.ZnDMdaCxBbT4OMTv.JournalEntryPage.6bxwbC9bNF6FxSBS#mode-debug]{Mode Debug}, sur chaque feuille d'objet, dans l'onglet Détails apparaitra un cadre Debug avec :

  • ID : c'est une chaine de caractère unique permettant de différencier les éléments créés dans FoundryVTT. Si vous créez un Équipement "Dague" présent dans les dossiers de l'onglet Objets de FoundryVTT et que vous distribuez cette même "Dague" à chaque Personnage, chaque Objet "Dague" aura une ID différente et chaque joueur pourra modifier sa "Dague" sans que ça modifie l'Équipement des autres ni celui du dossier dans la barre latérale de FoundryVTT.

  • Clé : cette chaine de caractère n'est pas unique contrairement à l'ID. Elle permet de connaitre la désignation de l'Objet sans caractères spéciaux (espaces, majuscules…). Ca peut servir surtout pour la configuration de macro. Par exemple si vous souhaitez créer une macro ne fonctionnant que pour un Personnage ayant une "Épée à deux mains", il faudra utiliser la Clé pour identifier cet Objet ("epee-a-deux-mains").

Dans la mesure du possible, évitez d'activer le Mode Debug sauf dans des cas très précis. Même si l'ID et la Clé ne sont pas modifiable par mégarde, ces informations risques de perturber vos joueurs alors qu'elles ne leur servent à rien.

Feuille Profil

Dans le Compendium, tous les profils du Livre de Règles ont été disposés dans les quatre dossiers de Familles. Vous pourrez y trouver le Profil qui correspond à votre personnage et le glisser-déposer sur votre @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV]{Feuille Personnage}. Un Profil ne peut pas être associé à une @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille Rencontre}.

Si vous souhaitez créer un nouveau Profil ou modifier les Profils existants, vous trouverez ci-dessous l'explication pour chaque paramètre d'une feuille Profil.

Famille

La Famille permet de choisir parmis les 4 familles de Profil "Aventurier", "Combattant", "Mage" ou "Mystique". La famille détermine les valeurs de "Points de Vigueur par Niveau" et de "Dé de Récupération".

Point de Vigueur par Niveau

Cette valeur n'est pas modifiable. Elle est complétée lors du choix de la famille du Profil. Cela représente le nombre de PV reçu par le Personnage à chaque niveau.

Si vous avez un Profil Hybride, les PV sont calculés différemment. Au passage de niveau d'un Profil Hybride, vérifiez que le bon nombre de PV sont gagnés sinon il faudra modifier manuellement les PV du Personnnage dans la barre latérale de la feuille. Voir @UUID[JournalEntry.ap0bJTU9nAG3YfQn.JournalEntryPage.1lWZUPGyYMWZAjHa#points-de-vigueur]{Points de vigueur} pour plus d'informations.

Dé de Récupération

Cette valeur n'est pas modifiable. Elle est complétée lors du choix de la famille du Profil. Elle représente le type de dé utilisé lors d'une Récupération Rapide ou d'une Récupération Complète.

Défense max de l'armure

Indiquer ici la valeur maximale de l'armure que peut porté le Personnage ayant ce profil pour utiliser les capacités apprises dans les Voies de ce Profil. Ce chiffre est très important dans le système car si le Personnage équipe une armure ayant une défense supérieure à cette valeur, toutes les actions venant des capacités apprises par le Personnage ne seront plus visibles dans la liste des actions de l'@UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#onglet-principal]{Onglet Principal}.

Profil principal

Cette case permet de déterminer si ce Profil est le profil principal du Personnage ou non. Lorsque l'on ajoute le premier Profil à une feuille de Personnage, il devient automatiquement le Profil principal. Le deuxième profil ajouté ne le sera pas.

Si vous souhaitez changez il faudra ouvrir les feuilles de Profil dans le @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#cadre-profil-andtraits]{Cadre Profil &Traits} de l'onglet Description.

ATTENTION : il est possible de cocher la case sur les deux profils.

Modificateurs

En cliquant sur le bouton "+", il est possible d'ajouter des modificateurs spécifiques pour le Profil. Que ce soit un bonus ou un malus de caractéristique, une vision dans le noir, une immunité quelconque, une augmentation ou diminution des Points de Magie… on peut ajouter autant de Modificateurs que nécessaire.

Voir @UUID[JournalEntry.veSZmLHWcmklyKJJ.JournalEntryPage.FwpATHAaqOAEsHEn]{Modificateurs} pour plus d'informations.

Formations Martiales 

Cette section permet de sélectionner les Armes, Armures et Boucliers maîtrisés par le Profil. Les formations martiales sont séparées en trois catégories pour s'y retrouver plus facilement. Lorsque la feuille Profil est verrouillée, seules les formations apprises par le Profil sont visibles. Il faut déverrouiller la feuille pour en sélectionner des nouvelles ou en désélectionner certaines.

Voies

En glissant-déposant des Voies depuis la barre latérale de FoundryVTT, il est possible de rajouter les 5 voies (ou moins, ou plus) du Profil. Même si cela n'a que peut d'impact, les Voies sont triées dans l'ordre où elles sont glissées sur la fiche du Profil. Toutes les voies présentes sur le Profil seront ajoutées à la feuille Personnage lorsque le Profil sera glissé-déposé dessus.

Si un Personnage ne souhaite pas avoir toutes les voies configurés dans un Profil, il peut soit supprimer des voies dans l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités} en déverrouillant sa feuille soit en ouvrant la feuille de son Profil et en supprimant certaines voies (la feuille doit être déverrouillée). Les deux méthodes sont identiques. Utilisez l'icone "Poubelle" pour supprimer une Voie d'un Profil.

Feuille Trait

Le Trait est un Objet assez spécial qui permet de créer des Peuples ou une sorte de Profil spécifique. Pour les Peuples, cela permet de limiter à un seul le nombre de Peuple que l'on peut ajouter à une feuille Personnage. 

Pour les traits, cela donne la possibilité aux MJs de créer des archétypes facilement duplicables pour certains PNJ. Par exemple si dans votre campagne, vous comptez créer une grande lignée de Vampires, vous pourrez créer un Trait Vampire et renseigner les propriétés spécifiques pour cette "Espèce". Quelles sont les bonus ou les malus de caractéristiques des vampires, leur capacités et les voies qu'ils peuvent développer. Si un de vos PJ se fait transformer en Vampire durant la campagne, glisser-déposer sur sa feuille Personnage le Trait "Vampire" et en une action, il obtiendra les propriétés de cette "Espèce".

Pour simplifier, les Traits (hors Peuples) permettent de créer tous les types d'éléments qui ne sont ni une Voie, ni un Profil, ni une Capacité. 

ATTENTION : les Traits, de Peuple ou non, ne peuvent pas être ajoutés à une @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille Rencontre}. Seuls les @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV]{Feuille Personnage} peuvent recevoir un. Trait

Sous-type

Le sous-type permet de choisir le type de Trait parmis "Peuple" ou "Trait". Il est possible d'ajouter un seul Peuple sur une feuille de Personnage donc si vous voulez créer un élément qui se rajoute en plus d'un Peuple il faudra choisir Trait dans le Sous-type du Trait.

Modificateurs

En cliquant sur le bouton "+", il est possible d'ajouter des modificateurs spécifiques pour le Profil. Que ce soit un bonus ou un malus de caractéristique, une vision dans le noir, une immunité quelconque, une augmentation ou diminution des Points de Magie… on peut ajouter autant de Modificateurs que nécessaire.

Voir @UUID[JournalEntry.veSZmLHWcmklyKJJ.JournalEntryPage.FwpATHAaqOAEsHEn]{Modificateurs} pour plus d'informations.

Il est possible de supprimer un Modificateur en cliquant sur l'icone "Poubelle" lorsque la feuille est déverrouillée.

Voies

En glissant-déposant des Voies depuis la barre latérale de FoundryVTT, il est possible de rajouter une ou plusieurs Voies au Trait. Même si cela n'a que peut d'impact, les Voies sont triées dans l'ordre où elles sont glissées sur la fiche du Trait. Toutes les voies présentes sur le Trait seront ajoutées à la feuille Personnage lorsque le Trait sera glissé-déposé dessus.

Si un Personnage ne souhaite pas avoir toutes les voies configurés dans un Trait, il peut soit supprimer des voies dans l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités} en déverrouillant sa feuille soit en ouvrant la feuille de son Trait et en supprimant certaines voies (la feuille doit être déverrouillée). Les deux méthodes sont identiques. Utilisez l'icone "Poubelle" pour supprimer une Voie d'un Trait lorsque la feuille est déverrouillée.

Capacités

En glissant-déposant des Capacités depuis la barre latérale de FoundryVTT, il est possible de rajouter une ou plusieurs Capacités au Trait. Même si cela n'a que peut d'impact car il n'y a pas de notion de Rang sur un Trait, les Capacités sont triées dans l'ordre où elles sont glissées sur la fiche du Trait.

Les Capacités présentes sur la feuille de Trait seront ajoutées aux Capacités Hors-Voies de la feuille de Personnage. Elles seront directement apprises par le Personnage mais il sera possible de les supprimer depuis l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités} ou en ouvrant le feuille du Trait depuis l'@UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#onglet-biographie]{Onglet Biographie}.

Feuile Voie

Dans le compendium du Livre de Règles de Chroniques Oubliées Fantasy v2, vous trouverez toutes les Voies configurées présentes dans le livre. Vous pourrez trouver la ou les voies pour la création de votre Personnage et les glisser-déposer sur la feuille.

Dans la majorité des cas, les Voies seront ajoutées à des Profils et vous ajouterez le Profil à votre @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV]{Feuille Personnage} mais dans certains cas, il sera possible d'ajouter directement une Voie à la feuille d'un PNJ ou des protagoniste spéciale.

ATTENTION : comme pour les Traits et les Profils, les Voies ne peuvent pas être ajoutées à des @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille Rencontre}. Seules les @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV]{Feuille Personnage} peuvent recevoir des Voies.

Si vous souhaitez créer une noucelle Voie ou modifier une Voie existente, vous trouverez dans cette section toutes les informations nécessaire pour paramètrer une Voie.

Sous-type

Ce champs permet de choisir le type de Voie parmis les 5 types disponibles. Sauf pour le Sous-type "Voie de Prestige", les autres n'ont pas de différence à part pour s'y retrouver facilement lorsqu'on ouvre une Voie. Il est ainsi possible de savoir rapidement de quel type de Voie vous avez affaire.

Dans de future mises à jour du système, il sera possible de rajouter une Voie de créature à une Feuille rencontre.

Voie de Prestige

Le sous-type Voie de prestige est différent des autres car il apporte des changements.

Comme dans les règles (@UUID[JournalEntry.ap0bJTU9nAG3YfQn.JournalEntryPage.3e6HLmyaIDQ4DdPN]{Chapitre 8 : Voies de prestige}), les valeurs des Rangs des voies de Prestige vont de 4 à 8 et vous devez, en plus indiquer le nombre de Points de Vigueur par niveau obtroyé par la Voie de Prestige. Lors d'un passage de Niveau si le Joueur apprend une capacité de sa Voie de prestige, il obtiendra le nombre de PV inscrit dans cette case.

De plus, il n'est possible d'ajouter qu'une seule Voie de Prestige sur une feuille Personnage

Rang

Cette valeur n'est pas modifiable. Elle permet de connaitre le Rang de la plus haute capacité apprise par le Personnage qui possède cette Voie. Cette valeur est utile pour de nombreux calculs de capacité ("gagner 2 + rang en discrétion", par exemple).

Défense maximale de l'armure

Indiquer ici la valeur maximale de l'armure que peut porté le Personnage ayant cette Voie pour utiliser les capacités apprises dans la Voies. Ce chiffre est très important dans le système car si le Personnage équipe une armure ayant une défense supérieure à cette valeur, toutes les actions venant des capacités apprises par le Personnage ne seront plus visibles dans la liste des actions de l'@UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#onglet-principal]{Onglet Principal}.

Si la Voie fait parti d'une @UUID[JournalEntry.3Kelbp4V9f2yddBb.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille PROFIL}, la Défense maximale de l'armure de la Voie sera configurée à 0 car c'est la valeur configurée dans le Profil qui prendra le dessus. Par contre certaines capacités de COF permettent d'utiliser les capacités de certaines Voies sans tenir compte de la limitation d'armure du Profil. Dans ce cas, vous pourrez modifier la valeur sur la feuille de la Voie en conséquence.

Capacités 

En glissant-déposant des Capacités depuis la barre latérale de FoundryVTT, il est possible de rajouter une ou plusieurs Capacités à la Voie. Les Capacités seront triées dans l'ordre où elles sont glissées sur la fiche de la Voie. Cela est particulièrement important à respecter pour que chaque capacité ai le bon Rang dans la Voie.

Les Capacités ajoutées à une Voie ne sont pas automatiquement apprises lorsque la Voie est glissée-déposée sur une feuille Personnage contrairement aux Capacités ajoutées aux Traits.

Même lorsque la Voie est verrouillée, il est possible d'ouvrir les feuilles Capacités en cliquant sur l'image de la Capacité afin de lire son effet. 

Quand la feuille est déverrouillée, utilisez l'icone "Poubelle" pour supprimer une Capacité ajoutée à la Voie.

Feuille Capacité

Comme son nom l'indique, le type d'Objet "Capacité" permet de créer une nouvelle Capacité qu'il sera possible d'intégrer ensuite dans une @UUID[JournalEntry.3Kelbp4V9f2yddBb.JournalEntryPage.FwpATHAaqOAEsHEn]{Feuille VOIE} puis dans une @UUID[JournalEntry.3Kelbp4V9f2yddBb.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille PROFIL} ou de la laisser indépendante pour les capacités Hors Voies, par exemple.

Les Capacités sont la base des actions des Personnages. En apprenant des Capacités, il obtiendra de nouveaux pouvoirs, des facultés nouvelles ou des sorts puissants.

Il est possible d'ajouter une Capacité sur une @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.yLgtzWasiKKcYUaV]{Feuille Rencontre} et une @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV]{Feuille Personnage}. Les Capacités pour les Personnages et les Rencontres sont exactement les mêmes et se configurent de la même manière.

Si vous ajouter une Capacité directement sur un Acteur en glissant-déposant la Capacité sur la feuille, elle apparaitra dans la liste des Capacités Hors-voies et elle sera directement apprise par le Personnage ou la Rencontre. De plus, elle aura la valeur de Rang de sa place dans la Voie d'où elle est extraite.

Dans le Compendium du Livre de Règles, vous retrouverez toutes les @UUID[Compendium.cof-base.cof-2-base-items.Folder.bIBuYZXCH4B9OUNH]{Capacités} décrites dans les Voies de Profil mais aussi toutes les @UUID[Compendium.cof-base.cof-2-base-items.Folder.Jc9oocmI3B4cVSRG]{Capacités des rencontres}.

Si vous souhaitez créer une Capacité ou modifier une existente, voici le détails des paramètres pour les Capacités.

Rang

Ici est noté la valeur du rang de la Capacité dans la Voie. Cette valeur sert à calculer de nombreuses variables pour les Actions du Personnage ou de la Rencontre ("2 + rang en diplomacie", par exemple). Si la Capacité est intégré dans une Voie, vous n'aurez pas besoin de modifier cette valeur. Par contre si vous créez une Capacité vous pourrez indiquer sa valeur du Rang.

Coût en Points de Capacité

De base, configuré à "-1", cette ligne est modifable seulement lorsque la fiche est déverrouillée. En laissant à "-1" le coût en Points de Capacité sera automatiquement calculé en fonction du rang de la Capacité (1 point pour les rangs 1 et les rangs 2 et 2 points pour les rangs 3, 4 et 5).

Il est toutefois possible de modifier cette valeur pour créer des capacités qui ne coûtent aucun point de capacités ou au contraire d'augmenter leur coût par rapport aux règles du Livre de Règles. La plupart du temps, ce champs sera utile pour créer des Capacités Hors-voies qui ne coûtent aucun Points de Capacité. Pour créer un Objet prestigieux (un objet magique qui intègre une Voie propre @UUID[Compendium.cof-base.cof-2-base-regles.JournalEntry.gKBkD8f28NlLwsZA.JournalEntryPage.AnLhalcoTGGMEn1w#objets-prestigieux]{Objets prestigieux}), il se pourrait, par exemple, que vous ayez besoin de modifier la valeur du Coût en Points de Capacité pour les augmenter ou les diminuer.

Apprise

Cette case à cocher permet de définir si la Capacité est apprise ou non par le Personnage lorsque la capacité est déjà sur une feuille.

A la création de la capacité si la case est déjà cochée, la capacité sera apprise automatiquement lorsqu'elle sera glissée-déposée sur une feuille. Lors d'une dépense de Points de Capacité, après un passage de Niveau, les Joueurs peuvent apprendre leurs nouvelles capacités dans l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités} ou en ouvrant les feuilles Capacités et en cochant cette case lorsque la feuille est déverrouillée.

Type d'Action 

Choisir dans cette liste déroulante le Type d'Action de la capacité entre "Action Limité", "Action de Mouvement", "Action d'Attaque" et "Action Gratuite". Cela permet d'afficher une lettre (L, A, M ou G) sur la ligne de la capacité.

Le type d'action est général et s'applique pour toutes les Actions et Effets ajoutés dans l'onglet @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.a0cY3q3XV4o8Y13V]{Actions et Effets}. 

Pour le moment, le temps que prend une action pour être réaliser n'a pas d'importance dans le système mais nous espérons intégrer dans les prochaines mises à jour un système pour compter les actions réalisables par les Personnages. Ainsi, en plus d'être une information importante pour les joueurs, nous vous conseillons de bien définir ce champs.

Fréquence d'Utilisation

Si le nombre d'utilisation de la capacité est limité, il est possible de le renseigner sur cette ligne. Cela aura un impact sur la liste d'Actions du Personnage dans son onglet Principal dans la liste du @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#cadre-actions]{Cadre Actions}. Si la capacité n'a pas de Fréquence d'Utilisation, elle pourra être utilisé autant de fois que voulue.

Si la fréquence est définie par "Combat", il faudra réaliser une @UUID[JournalEntry.9e6hXE2pA2Tdprd1.JournalEntryPage.Dw9Ze41hgZ5MdKg2#recuperation-rapide]{Récupération Rapide} pour récupérer les utilisations de la capacité. Si la fréquence est définie par "Qotidien", il faudra réaliser une @UUID[JournalEntry.9e6hXE2pA2Tdprd1.JournalEntryPage.Dw9Ze41hgZ5MdKg2#recuperation-complete]{Récupération Complète} pour récupérer le maximum des utilisations de la capacité.

Nombre d'Utilisation

Ces champs permettent de définir le nombre maximum et actuel d'utilisation de la capacité par le personnage. 

Pour le moment, dans le champ "Maximum" il n'est possible de renseigner qu'un nombre. Vous ne pouvez pas inscrire une variable de votre personnage pour que le nombre évolue en temps réel. Vous devrez penser à modifier le nombre maximum d'utilisation de la capacité quand vos modificateurs de caractéristiques évoluent, par exemple.

La valeur Actuelle d'utilisation est réduite quand le Personnage ou la Rencontre utilise la capacité et augmente en fonction des Récupérations effectuées. Il est possible, bien entendu, de modifier manuellement cette valeur en ouvrant la feuille Capacité sans avoir besoin de la déverrouiller pour plus de rapidité.

Sort

cette case permet de définir si la capacité est un sort ou non.

Si vous cocher la case Sortilège, cela fera apparaitre un champ pour définir le coût de mana du Sort. En utilisant une Capacité définie en Sort, le Nombre de Points de Mana inscrit dans le champ sera déduit du nombre de PM du Personnage. 

Les Capacités configurées en Sort sont repérées par une * à la suite de leur nom dans l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités} et dans le @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#cadre-actions]{Cadre Actions}. 

Coût en Mana

Cette information n'apparait QUE pour les capacités définies comme "Sorts" comme mentionné précédemment.

De base, configuré à "-1", cette ligne est modifiable seulement lorsque la fiche est déverrouillée. En laissant à "-1", le coût en PM de la capacité sera calculé automatiquement en fonction du rang de la capacité. Il est toutefois possible de modifier cette valeur pour créer des capacités ayant des coût en Points de Magie qui ne respecte pas cette règle. 

Le coût en Points de Mana apparait à la suite du nom de la capacité dans la liste des Actions dans le @UUID[JournalEntry.pIFqDS5WsMoOdlbC.JournalEntryPage.wnsowXuYD15fMeGV#cadre-actions]{Cadre Actions} de la feuille Personnage.

Nous détaillerons cela dans le prochain journal mais il est possible de rajouter des @UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.a0cY3q3XV4o8Y13V]{Actions et Effets} sur une Capacité configurée en Sort et de paramètrer chaque Action pour renseigner si cette Action spécifique consomme le coût en Points de Mana ou non.

Autoriser une capacité liée

Certaines capacité du Livre de Règles permettent d'apprendre des capacités venant d'autres Voies. C'est grâce à cette case à cocher que vous pourrez activer cet possibilité.

Lorsque la case est cochée, un champs "Capacité liée" apparait sur la feuille de la Capacité. De plus, dans l'@UUID[Compendium.cof-base.cof-2-guide-du-systeme.JournalEntry.Cj2L7dqSXItRSNXi.JournalEntryPage.9Bn7F4sxyKm3TX74#onglet-voies-et-capacites]{Onglet Voies et Capacités}, une nouvelle ligne apparait sous cette capacité lorsqu'elle est apprise.

  • Si la Capacité permet de choisir une autre capacité au moment où elle est apprise, ne renseignez rien dans le champs "Capacité liée" à la création. Il suffira au joueur de glisser-déposer la capacité qu'il a choisi sur la ligne en dessous de la capacité apprise pour l'ajouter. Aucune limitation ne peut être définie dans ce champ donc vérifiez que le texte de la Capacité est respecté.

  • Si la Capacité stipule la capacité bonus lorsque celle-ci est apprise, vous pouvez renseigner cette capacité dans le champ "Capacité liée". Ainsi, automatiquement, à l'apprentissage de la Capacité "mère", la Capacité "fille" sera ajoutée en dessous. Pour ajouter une capacité liée, vous pouvez copier-coller l'ID de la capacité choisie dans le champ et appuyer sur le bouton "ajouter document". Sinon, cherchez la capacité à lier et glissez-déposez la sur la feuille de la Capacité "mère". Dans les deux cas, le nom de la Capacité choisie apparaitra juste au dessus.
